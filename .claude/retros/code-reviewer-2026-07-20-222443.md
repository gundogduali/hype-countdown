## Lesson learned
Added to `.claude/agents/code-reviewer.md`: before treating any same-port/
same-command-name process as "my own orphan, safe to kill," verify actual
ownership via `ps eww -p <pid>` (env vars) or `lsof -p <pid>` (open DB file
path) — job-control state (`%1`) does not persist across separate Bash tool
calls, so a `kill`+immediate `ps aux` re-check in this environment can be
unreliable in both directions, and a *different* concurrent agent's process can
legitimately end up bound to the exact port you were told to use.
