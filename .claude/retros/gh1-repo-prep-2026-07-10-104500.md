## What the review found and fixed
- First stills caught the NumberFlow seconds digit mid-slide (ghosted half-digits) — looked like a rendering bug in a static image. Retook all stills with Playwright `animations: 'disabled'` (fast-forwards finite WAAPI animations), digits fully settled. The GIF intentionally keeps animations enabled — the slide is the point.
- First GIF pass (960px/128 colors/bayer) was only 202 KB; budget allowed a higher-quality re-encode (1200px/256 colors) at 343 KB.
- `git rev-parse --abbrev-ref HEAD` errors on a zero-commit repo; used `git symbolic-ref --short HEAD` instead.

## Lesson
- When taking static screenshots of continuously animating UI, always pass `animations: 'disabled'` (or wait for a settled window); keep animations enabled only for motion captures. (No agent definition file of my own to record this in — noting here for frontend-developer/qa-engineer.)
